# UI Refresh — Identity, Clarity, Jump-Rail, and Touch Foundation

A coordinated refresh of the chat-facing UI across two design passes. Pass 1 (decisions 1–7) surfaced the existing brand mark, made the evidence/settings/sidebar/compaction surfaces legible, and added a conversation jump-rail. Pass 2 (decisions 8–14) followed a grounded UX audit of the knowledge, composer, and mobile surfaces: it establishes a shared touch/capability model, fixes real bugs (mobile compare, silent failures, garbage text), and removes friction (FileProductionCard revamp, composer dead-ends, favicon privacy leak). The mark's visual design and animation are deliberately left unchanged throughout.

## Decisions

**1. LogoMark: unchanged look, expanded placement.** The mark keeps its existing beige `#C8A882` color and 6-stage stroke-draw animation — no recolor, no new motion. Only its *presence* changes: it is added to the sidebar header (beside the wordmark), the collapsed sidebar rail (as the rail icon), the empty conversation state (as a static centerpiece), the mobile header, and the browser favicon. It remains under the last generating message. Rejected: recoloring the mark to terracotta to match the UI accent (loses the mark's quiet character); adopting the gold as a new global accent (gold is lower-contrast than terracotta, weakening CTAs); a two-tone treatment (introduces a second color to manage with no clear rule). The beige-mark / terracotta-chrome split is accepted as intentional.

**2. Evidence tab renamed "Sources" — informational only.** The per-message disclosure labeled "Evidence" becomes **"Sources"** with a Lucide `Book` icon. Its purpose is transparency: it answers "what did the model read, and what did it ignore?" for the current answer. Two visual groups — **Used** (terracotta-accented, with Lucide type icons: `FileText`/`Globe`/`Quote`/`Paperclip`) and **Set aside** (muted) — replace the flat row list. Documents are clickable and open the DocumentWorkspace; web links keep a terracotta-tinted underline. The Auto/Pinned/Excluded steering control is **removed from the chat surface entirely** and lives in the Knowledge library / working-document workspace where documents are already managed. Favicons are reused from the existing `ThinkingBlock` favicon path. "Reranked N%" is no longer shown. Rejected: making Sources a steering control panel (purpose drift from transparency); citations-only inline `[n]` legend (less room for the "considered" transparency). The `_Avoid: context sources` note in CONTEXT.md is reconciled by keeping **Context Sources** as the concept name and **Sources** as the chat-surface label.

**3. Sidebar S1 — same structure, unified rules.** Keeps the three sections (Pinned / Projects / Chats) but fixes the rule inconsistencies: **all three are collapsible**, each header has an icon + label (Pinned=`Pin`, Projects=`Folder`, Chats=`MessageSquare`), the **Chats header never vanishes** (empty state lives inside it), project folders show **chat-count badges**, touch/hybrid devices get **always-visible affordances** (no `md:opacity-0` hover-gating of primary actions), the collapsed rail shows an **active-conversation indicator**, **Logout requires confirmation**, and sidebar/header-menu **vocabulary is unified**. Rejected: merging Pinned into Recent (loses the explicit pin mental model); time-bucketing like Codex/ChatGPT (biggest departure, loses project folders as primary grouping); dropping the Projects section for filter pills (loses the folder-tree model power users rely on).

**4. Conversation jump-rail — floating, conditional, cursor-relative.** A new vertical rail floating over the left edge of the chat, vertically centered, not occupying layout space. One thin mark per turn (height encodes content length); the active/viewport turn is thicker + terracotta. **Fades in after 6 messages** (0.6s ease-out, 6px slide) and is absent on shorter conversations. Hover reveals an H2-style snippet of the assistant reply in serif (no turn number — irrelevant), and the marks animate a cursor-relative scale + color sweep wave originating from the hovered mark. **Hidden on mobile** (no hover; would collide with content). Under `prefers-reduced-motion`, the fade-in is instant and the wave is skipped (the active-line state is preserved, being state not motion). Rejected: an always-occupying sidebar column (wastes space); a minimap with a tinted viewport window (redundant — the active line already signals position); turn-number labels (irrelevant to the user). Note: this is a pure client-side navigation affordance over the existing `MessageArea` message list; it does not change message persistence, streaming, or the conversation-detail read model.

**5. Settings Profile rebuild — grouped, icon buttons, Skills promoted, analytics merged.** The flat 7-card Profile tab becomes **5 grouped sections**: Account (avatar + display name + email + password + import, merged), Preferences (default model, conversation style [was "Default Style"], appearance, language [UI + title, clarified]), Assistant (Skills as a summary card → dedicated full-screen manager), Data & privacy (download / clear / clear / delete with distinct destructive styling), and **Your Activity** (the 5 personal analytics stats + 2 charts, merged from the now-removed Analytics tab). All text CTAs become canonical `btn-icon-bare` Lucide icon buttons (`ChevronRight` to drill in, `Eye`/`RotateCw`/`Download`/`Trash2` for actions; solid red `Trash2` for account deletion — the first strong CTA on the page). **The top-level Analytics tab is removed for normal users** (merges into Profile); **system analytics stays admin-only and moves under Administration** as a third pane alongside System and Users. Server-side gating already omits system/per-user data for non-admins, so this is a tab-structure change, not a data-flow change. Rejected: promoting Skills to its own top-level settings tab (over-rotates a niche feature); leaving Skills inline (breaks visual scale); keeping a separate Analytics tab for normal users (the personal stats are more discoverable inside Profile).

**6. Compaction UI — human words, chip-divider marker, silent popover during compaction.** Two surfaces. (a) The **ContextUsageRing popover** is rewritten in plain language: a **"Context room"** bar (no redundant "plenty left" sub-text), **"What AlfyAI remembers"**, **"Sources included"**, **"Memory in play"**, plus a one-line explainer of what happens at capacity. Cost is shown as two compact lines: **conversation total + last turn**, each with cost + token count on one line ("$0.1840 · 412k tokens"). A **near-trigger heads-up** appears around 78% usage (bar shifts amber, a one-line note explains older messages will be summarized). The popover is **silent during compaction** — it does not show in-progress state; it updates once when compaction finishes. (b) The **inline compaction marker** becomes a **C1 chip-divider**: a centered pill between two hairline gradient lines, gold-tinted, sitting between messages like an editorial section break. In progress, the pill's color breathes (no loading icon — the color change is the signal). Done: the pill is static, labeled "Summarized N earlier messages," with an `Eye` icon button that expands the LLM-generated summary inline (the digest already persisted in message metadata — no new data) and a line clarifying originals are still saved. Failed: red-tinted pill with a `RotateCw` retry icon button. Rejected: the prior left-bordered-strip layout (boxy, didn't feel intentional); full-width banners / gutter tabs / near-invisible hairline variants (each rejected for being too loud, too marginal, or too cryptic respectively); showing in-progress state in the popover (the inline marker already carries it; duplicating in the popover is noise).

**7. Empty conversation state — static brand moment.** Replaces the dead "Conversation Ready" text with the static (non-animated) LogoMark as centerpiece, one serif headline ("How can I help?" — a question, not a status), and a quiet hint pointing to the composer. No suggestion prompts. Rejected: suggestion prompts / starter cards (content to maintain, and the user wanted a pure blank-canvas feel); dynamic suggestions (engineering cost with no clear payoff); rotating tips (off-topic for a blank canvas).

## Pass 2 — Touch Foundation, Knowledge Library, and Chat-Surface Defects

**8. Shared touch/capability model.** Replace the 4+ inconsistent mobile checks with one canonical rule combining three width tiers — phone (`<640px`), tablet (`640–1023px`), desktop (`≥1024px`, matching the existing Header/Sidebar `lg:` breakpoint) — and a capability check (`(hover: none) and (pointer: coarse)` = touch device). The two combine: a touch tablet gets touch affordances at tablet-width layout; a touch laptop gets touch affordances at desktop layout; a narrow desktop window gets narrow layout but hover affordances. Rejected: width-only (ignores touch laptops/tablets); capability-only (ignores legitimate narrow-window cases).

**9. Hover-only affordances → touch + focus fallback.** On a touch device, every hover-revealed affordance (message action row, code copy, audit-info popover, action tooltips) becomes always-visible at ≥44px targets at all widths — not just `≥640px` as today. On desktop with hover, affordances stay quiet by default **but `:focus-visible` reveals them identically to hover**, so keyboard users are never locked out (today focus lands on invisible buttons). Code copy specifically shows at 40% opacity by default on desktop → 100% on hover (discoverable, not noisy). Popover-type affordances (info, tooltips) open via tap-toggle on touch and hover/focus + outside-click on desktop. Rejected: always-visible at full opacity everywhere (too noisy on desktop); keeping hover-only with no focus path.

**10. Mobile keyboard jank — viewport meta fix.** Add `interactive-widget=resizes-content` to the viewport meta in `src/app.html`, then delete the custom `keyboardOffset` JS and `padding-bottom` transition in `ChatComposerPanel.svelte` in favor of CSS `100dvh` + bottom anchoring. The browser now handles the keyboard natively; today the custom offset double-applies on Android and the 150ms transition makes the composer visibly lag on every keyboard show. Pure code change, no design surface.

**11. Knowledge Library fixes.** (a) Remove the stray `...` text node in the bulk-delete button (visible garbage bug). (b) Surface a toast/inline error when a dropped file is rejected (`validFiles.length === 0`) and validate size client-side before upload — today invalid/oversized drops vanish silently. (c) Drive the existing `.loading` opacity + an input spinner from a navigating flag around `updateKnowledgeLibraryParams` — today server-side search/sort/page changes show stale results with no feedback. (d) Add a "Select all (N)" control in the bulk-action bar so mobile users (thead hidden) can still select-all. (e) Remove the table's 600px inner `max-height` on desktop; let it grow with the page scroll, keep only the sticky header — today two scrollbars fight. (f) Memory category empty states get a one-line explainer ("Memories are captured from your conversations") + link to settings — today they're dead ends.

**12. "AI-facing version" → "What the AI sees".** The Bot-icon row action labeled "AI-facing version" is renamed "What the AI sees" (Eye icon, terracotta) with a one-line explainer: "AlfyAI reads this normalized text instead of the raw file — shown so you can verify the model understood your document correctly." Rejected: keeping "AI-facing version" (internal jargon with no explanation of why some rows have it).

**13. Memory actions — one "Remove" entry point, modal teaches the choice.** Replace the two indistinguishable instant-firing destructive icon buttons ("Do not remember" X + "Delete" trash) with a single trash entry point that opens a confirm modal: **"Remove this memory?"** with three options — **Forget** (EyeOff, suppress: "Stops AlfyAI from bringing this up on its own, but keeps the record"), **Delete permanently** (Trash, full removal: "Erases this memory completely"), **Cancel** + X top-right. The modal teaches the difference in plain language instead of hiding it. Rejected: keeping both as separate bare icons (users can't tell suppress from delete); collapsing to delete-only (loses the pause-and-resume capability of suppress).

**14. Search hint, FileProductionCard revamp, composer fixes, favicon privacy, mobile compare.** (a) **Search:** with <2 chars show "Keep typing to search…" instead of misleading recents; recents appear only when the query is empty. (b) **FileProductionCard running state:** the active branch is literally empty today (just shimmer + X). Wire up `job.title` (exists, not rendered) + a client-side elapsed timer from `job.createdAt` + an animated gold progress bar (reuses the compaction gradient-sweep motion language) + a quiet Stop cancel icon. No backend changes for the running state. (c) **Stale-job honesty:** if `createdAt` is >90s old and still "running" with no heartbeat, the card shifts to an amber "Still working… or stalled. We'll know in a moment." state (client-side heuristic only) — today it pretends everything's fine until the 10-min server stale-recovery flips it. (d) **FileProductionCard dismiss (the one backend change in this ADR):** add a persisted `dismissed` flag on `file_production_jobs` (new column + migration + read-model filter). Non-retryable failures and cancelled jobs get a Dismiss (X) button; dismissed cards stop resurfacing on reload. Restorable via a per-message "show dismissed" affordance. Today failed/cancelled cards persist forever with no way to remove them short of deleting the conversation. (e) **Failed non-retryable card:** human error ("Couldn't produce this file") + plain cause + Dismiss + a one-line suggestion (e.g. "Try asking AlfyAI to simplify the request"). (f) **Composer over-length paste:** show a red counter (`50,000 / 10,000 — too long to send`) always visible (not just above 80%), but **no truncation button** — silent cuts hide what's lost; the user edits their own content. (g) **Composer disabled-send reason:** when `canSend` is false but text is non-empty, show a one-line hint ("Waiting for attachment…") instead of a silent disabled button. (h) **Composer "/" shortcut:** pressing `/` anywhere outside an input focuses the composer. (i) **Favicon privacy:** replace the direct Google S2 favicon call in `ThinkingBlock` with a same-origin proxy route `/api/favicon?domain=...` that fetches the source site's own `/favicon.ico` first, falls back to DuckDuckGo only if that 404s, caches the result — the browser never contacts either. Today every researched domain leaks to Google and the favicons break entirely offline. (j) **Mobile compare bug:** the mobile compare branch in `DocumentWorkspace.svelte` renders compared content twice (copy-paste bug); make it match the desktop branch (one body per section).

## Context

The UI was functionally complete but felt bland and, in places, confusing to its own developer. Pass 1's grounded audit (three parallel code explorations of the chat, sidebar, and settings/knowledge surfaces) surfaced concrete causes: the crafted animated LogoMark was buried in only two locations; the "Evidence" disclosure used ranker internals ("Reranked 87%", "Retrieved/Memory/Tool") as user-facing labels with no stated purpose; the sidebar had three sections with three different collapse rules and touch-invisible affordances; the Profile tab stacked seven visually identical cards including a 716-line Skills editor; the compaction marker and context popover spoke in jargon; and there was no way to navigate a long conversation except scrolling. Pass 2 audited the surfaces Pass 1 didn't touch (Knowledge Library, composer, message rendering, mobile, document workspace) and found a consistent pattern: hover-only affordances with no touch or keyboard path; 4+ incompatible "is this mobile?" checks causing tablet-band chaos; a mobile keyboard handler fighting the browser; silent failures (rejected drops, stalled jobs, dead-end error cards); and a privacy leak to Google via favicons. Each decision addresses a specific diagnosed problem while preserving existing boundaries.

## Consequences

- **Pass 1: no backend/data changes.** Decisions 1–7 are client-surface or i18n changes. The jump-rail reads the existing message list; Sources reads existing evidence data; compaction summary expansion reads existing message metadata; the cost/token popover values already flow to the ring.
- **Pass 2: one backend change — the file-production dismiss flag.** Decision 14(d) adds a `dismissed` column to `file_production_jobs` (+ migration + read-model filter). This is the only persistence change in the whole ADR; it directly enables dismissed error/cancelled cards to stop resurfacing on reload. Everything else in Pass 2 (running-card title/elapsed, stale heuristic, composer fixes, search hint, favicon proxy, mobile compare) is client-only or a new thin route.
- **New route: `/api/favicon`.** Decision 14(i) adds a same-origin favicon proxy route. It must be unauthenticated-safe (no auth gate on a favicon lookup), cache aggressively, and fail gracefully to a generic globe icon.
- **i18n: new and renamed strings.** "Sources" replaces "Evidence"; new compaction copy; new Profile section headers; new empty-state copy; "What the AI sees"; memory modal copy (Forget/Delete permanently/Cancel); FileProductionCard running/failed copy; composer over-length + disabled-send hints; search "Keep typing…". All new user-facing strings must be localized in English and Hungarian per the existing rule.
- **One component split.** The Skills editor becomes a dedicated full-screen manager reached from a Profile summary card; `UserSkillsSettingsSurface.svelte` content is largely unchanged, only its host changes.
- **Analytics tab removal is tab-structure only.** `SettingsAnalyticsTab.svelte` Block A (personal) renders inside Profile's "Your Activity"; Blocks B/C/D (admin) move under Administration. `getAnalyticsDashboardReadModel` and `/api/analytics` are unchanged.
- **CONTEXT.md reconciliation.** The `_Avoid: context sources` note is reconciled: **Context Sources** remains the concept; **Sources** is the chat-surface label. Pin/exclude steering statements move from the chat-surface context to the Knowledge/working-document context.
