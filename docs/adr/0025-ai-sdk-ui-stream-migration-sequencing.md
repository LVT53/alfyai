# AI SDK UI streams and messages ship before the next remote deployment

Accepted. AlfyAI will retire Langflow first by implementing **Normal Chat Model Run** on top of Vercel AI SDK, then replace the browser-facing stream contract and Normal Chat message runtime shape with Vercel AI SDK UI streams and UI messages in separate local development passes. The browser-facing stream contract has moved from the existing **Browser SSE Protocol** to AI SDK UI stream framing. Normal Chat persistence still uses the app-owned message schema and remains a follow-up before the next remote deployment if we want durable messages to match AI SDK UI messages directly. The next remote deployment should happen only after Langflow is removed, Vercel AI SDK Normal Chat is stable, AI SDK UI streams are stable, and either UI-compatible persistence is complete or explicitly accepted as a temporary app-owned envelope.

**Considered Options**

- Replace Langflow and the browser stream protocol in one migration.
- Keep the Browser SSE Protocol permanently and treat AI SDK UI streams as irrelevant.
- Retain AlfyAI's existing durable chat message schema and translate AI SDK UI streams at the runtime boundary.
- Retire Langflow locally first, then migrate to AI SDK UI streams and UI messages as a separate browser/runtime pass before remote deployment.

We chose the staged local migration because AI SDK UI streams and UI messages reduce long-term custom protocol and translation ownership and align better with Vercel's UI tooling, but switching stream/message contracts while replacing Langflow would combine two high-risk changes: model runtime behavior and browser chat runtime behavior. During the first local pass, AI SDK stream and tool parts were adapted into the existing runtime to complete Langflow removal. During the stream pass, AI SDK UI streams intentionally superseded ADR-0016 for the browser-facing contract while preserving app-owned active-stream lifecycle and durable message persistence. Normal Chat persistence may still move to an AI SDK UI message-compatible shape in a later pass. This private deployment may reset the database instead of carrying a legacy conversation migration if that produces a cleaner runtime before the next remote deployment; only configuration needed to bring the new runtime online, such as admin access, model/provider configuration, prompts, defaults, and still-relevant system skills, should be preserved or re-seeded. The stream pass needs tests for reconnect, stop/detach, generated files, context sources, message evidence, queueing, and durable completion metadata before deployment; the message-persistence pass needs equivalent persistence and reload coverage.
